# internal/cli

Argument-dispatch core shared by every verb. Defines the `Command` interface, the `Group` verb-tree node, the `IO` struct, the `Flagger` declaration interface, and a Cobra-style resolve-then-parse pipeline (`Resolve` + `Dispatch`) that walks argv against the verb tree, parses every flag against a single merged `*flag.FlagSet`, and hands off to the resolved leaf. Pure dispatch logic — no dependency on `internal/transport` or `internal/config`.

The flag pipeline mirrors mainstream "scoped flag set" CLIs (Cobra, Click, urfave-cli, clap): each `*Group` may declare persistent flags via its `Flags` closure that descendant leaves inherit; each leaf may declare local flags by implementing `Flagger`. Names a leaf re-declares automatically SHADOW the same name in any ancestor `Group.Flags` closure — the resolver registers the leaf's flags first on the merged FlagSet, then runs each ancestor closure on a private shadow set and copies only the non-clashing entries onto the merged set. That structurally prevents the "global flag stomps leaf flag" class of bug (e.g. `ana profile add --endpoint X` saving the default endpoint).

## Files

- `cli.go` — `Command`, `IO`, `DefaultIO`, `Flagger` (opt-in declaration), `Group` (verb-tree node with persistent `Flags`), `Group.Run` (re-enters `Resolve` so any Group can self-dispatch), `renderFlagsAsText` (sorted `--name <type>   usage (default: X)` enumeration), `Group.Help` (sorted child listing + trailing Flags block when persistent flags are declared), `IsHelpArg`, `FirstLine`. `dispatchChild` and the ancestor-flag-stack push are gone — the resolver subsumes both.
- `resolve.go` — the resolver: `Resolved` (Leaf + Path + MergedFS + Args), `Resolve` (walks argv with shape-aware flag-token consumption, identifies the leaf, then assembles MergedFS leaf-first and parses every flag), `Resolved.Execute` (runs the leaf or renders group help when only a group prefix was typed), `globalFromFlagSet` (extracts the four well-known root persistent flags into `Global`), `WithFlagSet` / `FlagSetFrom` (ctx helpers leaves use to fetch the parsed FlagSet for `RequireFlags` / `FlagWasSet`). `mergeShadow` is the Lookup-guarded copy that lets ancestor `Flags` closures use raw `fs.StringVar` / `BoolVar` / `IntVar` without panicking on names a leaf already declared.
- `dispatch.go` — `Dispatch` (root entry: prints root help on empty/help-token, otherwise `Resolve` → `Execute`; wraps usage errors with `ErrReported` so main()'s fallback printer doesn't double-emit), `RootHelp` (`root.Help()`), `RenderResolvedHelp` / `renderResolvedHelp` (group-leaf prints group help; leaf prints leaf help + merged Flags block — every flag in scope, leaf + ancestor persistent).
- `root.go` — `Global` (the four root persistent flags: `JSON`, `Endpoint`, `TokenFile`, `Profile`; populated by `globalFromFlagSet`), `WithGlobal` / `GlobalFrom` (ctx helpers for verb leaves; both panic on nil ctx per stdlib `context.Value` convention), `parseFlagToken` (classifies a token as long-form `--name`/`-name` with optional `=value`; both single- and double-dash spellings accepted per stdlib `flag.FlagSet.Parse`).
- `flags.go` — `ParseFlags` (interleaved-positional-tolerant wrapper around `flag.FlagSet.Parse`; the resolver uses it internally), `RequireFlags` (sorted `missing required flags: --a, --b` usage error; safely no-ops on a nil fs so leaves can call it from Run without guarding), `FlagWasSet` (the `fs.Visit` wrapper partial-update verbs use; nil fs reports false), and three typed `flag.Value` constructors: `EnumFlag` (allow-list), `IntListFlag` (CSV → `[]int`), `SinceFlag` (relative duration or RFC3339, UTC-normalised).
- `token.go` — `Token` (named `string` whose `String`/`Format` always render the `RedactToken` mask) plus its `Value()` escape hatch.
- `helpers.go` — verb-package helpers: `NewFlagSet` (rare cases that still build a private set), `UsageErrf`, `WriteJSON`, `Remarshal`, `RenderOutput` (JSON-vs-typed-render dispatch), `RequireStringID`, `RequireIntID`, `RenderTwoCol`, `ReadToken`, `ReadPassword`, `NewTableWriter`, `FirstLine`, `DashIfEmpty`, `RedactToken`.
- `errors.go` — `ErrUsage`, `ErrHelp`, `ErrReported`, `ExitCode` (maps sentinel chain + `auth.ErrNotLoggedIn` to the process exit code).
- `cli_test.go`, `flags_test.go`, `helpers_test.go`, `resolve_test.go` — cover dispatch, resolve, help rendering, flag parsing, exit-code mapping, and every branch of the shared verb helpers.
