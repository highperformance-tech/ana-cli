- 2026-04-21 ‚Äî `/connectors` Snowflake dialect CRUD (all 4 auth modes) ‚Äî +`api-catalog/POST_connector.create.snowflake.{password,keypair,oauth-sso,oauth-individual}.json`. Wire sub-object `config.snowflake`. Common fields: `locator` (Snowflake calls it `account`), `database`, `warehouse`, `role`, `schema`. Auth-mode discrimination is a BLEND of `authStrategy` top-level tag AND which credential field is populated in `config.snowflake`:
  - password ‚Üí `authStrategy="service_role"` + `username`+`password`
  - key-pair ‚Üí `authStrategy="service_role"` + `username`+`privateKey` (PEM string; passphrase via `privateKeyPassphrase`)
  - oauth-sso ‚Üí `authStrategy="oauth_sso"` + `oauthClientId`+`oauthClientSecret`
  - oauth-individual ‚Üí `authStrategy="per_member_oauth"` + same clientId/secret as oauth-sso (field set identical; only strategy tag differs). UI exposes an "SSO Mode" select (`Disabled`|`Direct SSO`|`Token Exchange`) ‚Äî not probed beyond default.
  Server accepts both OAuth modes in pending state (direct RPC POST returns 200 without handshake). UI gates Create button client-side via "Please complete OAuth authentication". New RPC service discovered: `snowflake_oauth.SnowflakeOAuthService/GetSnowflakeOAuthURL` (called when user clicks "Authenticate with Snowflake"); redirect URI is hardcoded to `app.textql.com/auth/snowflake/callback` ‚Äî CLI cannot complete handshake, oauth leaves ship instructional-only.
- 2026-04-21 ‚Äî `/connectors` Databricks dialect CRUD (all 4 auth modes) ‚Äî +`api-catalog/POST_connector.create.databricks.{access-token,client-credentials,oauth-sso,oauth-individual}.json`. Wire sub-object `config.databricks`. Common fields: `host`, `httpPath`, `port` (int), `catalog`, `schema`. Credential layout is a `databricksAuth` one-of variant (NOT split across top-level fields like Snowflake):

  - oauth-individual ‚Üí `authStrategy="per_member_oauth"` + SAME `databricksAuth.oauthU2m.{clientId, clientSecret}` variant as oauth-sso (only strategy tag differs).
